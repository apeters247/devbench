Get a Config Value with a Default Fallback — devbench cf --get --default

Read a key from any config file and return a fallback value when the path doesn't exist. One flag replaces fragile shell patterns and yq's // empty operator.

The Problem: Missing Keys Break CI Scripts

Getting a config value with a safe default in bash is ugly:

# Fragile: suppresses all errors, not just missing-key errors
TIMEOUT=$(yq '.server.timeout' config.yaml 2>/dev/null || echo 30)

# Even worse with jq
TIMEOUT=$(cat config.json | jq -r '.server.timeout // 30')

# yq's // empty syntax is easy to forget
TIMEOUT=$(yq '.server.timeout // "30"' config.yaml)

Each tool has a different syntax. All suppress errors broadly. None works across formats.

The Solution: devbench cf --get --default

# Return 30 when server.timeout is missing — exit 0 always
TIMEOUT=$(devbench cf config.yaml --get server.timeout --default 30)

# Works with any format
RETRIES=$(devbench cf config.toml --get retry.count --default 3)
DB_HOST=$(devbench cf settings.ini --get database.host --default localhost)
PORT=$(devbench cf .env --get APP_PORT --default 8080)

When the path exists: returns the value, exits 0.
When the path is missing: returns --default VALUE, exits 0.
No error suppression needed. Safe in set -e scripts.

GitHub Actions: Safe Config Extraction

# .github/workflows/deploy.yaml
- name: Read timeout config
  run: |
    TIMEOUT=$(devbench cf app-config.yaml --get server.timeout --default 60)
    WORKERS=$(devbench cf app-config.yaml --get server.workers --default 4)
    echo "TIMEOUT=$TIMEOUT" >> $GITHUB_ENV
    echo "WORKERS=$WORKERS" >> $GITHUB_ENV

No more || true workarounds. The deploy step never fails because a key was added later.

Kubernetes: Default Port When Not Configured

PORT=$(devbench cf values.yaml --get service.port --default 8080)
echo "Using port: $PORT"
kubectl expose deployment my-app --port=$PORT

Terraform: Default Region

REGION=$(devbench cf terraform.tfvars --get aws_region --default us-east-1)
terraform plan -var="aws_region=$REGION"

Raw Output for Shell Scripting

# --raw removes JSON wrapper — bare string for direct use
HOST=$(devbench cf config.yaml --get database.host --default localhost --raw)
echo "Connecting to: $HOST"
