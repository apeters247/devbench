Wrap YAML Under a Key from the Command Line


You have a flat values.yaml and need to nest the entire thing under spec.template.spec. Or you're generating a Kubernetes ConfigMap and need to wrap your app config under data. devbench cf --wrap-in does this in one command — no jq gymnastics, no Python script, no sed.


devbench cf config.yaml --wrap-in data
devbench cf values.yaml --wrap-in spec.template.spec --to json

Basic Wrapping


Wrap an entire config under a single key:


# config.yaml
host: localhost
port: 5432
name: mydb


devbench cf config.yaml --wrap-in database

database:
  host: localhost
  port: 5432
  name: mydb

Nested Dotted Path


Use dot-notation to create deeply nested output in one step. Each segment becomes an intermediate dict:


# values.yaml
replicas: 3
image: nginx:1.25
resources:
  limits:
    cpu: 500m
    memory: 512Mi


devbench cf values.yaml --wrap-in spec.template.spec

spec:
  template:
    spec:
      replicas: 3
      image: nginx:1.25
      resources:
        limits:
          cpu: 500m
          memory: 512Mi

Kubernetes ConfigMap Generation


Generate a partial Kubernetes ConfigMap patch by wrapping your config under data:


# Wrap app.yaml under data → ready for kubectl patch
devbench cf app.yaml --wrap-in data --to json | kubectl patch configmap app-config --patch-file /dev/stdin


# Pipe into envsubst → wrap → apply in one shot
envsubst < config.template.yaml | devbench cf - --wrap-in data | kubectl apply -f -

Helm Values Nesting


Helm charts often require values namespaced under a chart key. Instead of editing YAML manually:


# Wrap a sub-chart's values under its chart name
devbench cf backend-values.yaml --wrap-in backend

# Combine with --merge to overlay the wrapped values
devbench cf base-values.yaml --merge <(devbench cf backend.yaml --wrap-in backend)

Cross-Format: JSON Input → YAML Output


--wrap-in works on any of devbench's 11 supported formats. Wrap a JSON file and emit YAML:


devbench cf config.json --wrap-in metadata --to yaml

// config.json
{"name": "myapp", "version": "2.1.0"}


metadata:
  name: myapp
  version: 2.1.0

Terraform Module Wrapping


When building Terraform configs from templates, wrap your variable defaults under the module key:


# module defaults → wrapped under the module name
devbench cf defaults.yaml --wrap-in module.networking --to json > networking.auto.tfvars.json

Composing with Other Flags


# Wrap, sort keys, and convert to TOML
devbench cf config.yaml --wrap-in app --to toml --sort-keys

# Wrap and then merge with another file
devbench cf base.yaml --wrap-in config --merge overlay.yaml

# Wrap multiple files in batch
devbench cf --batch "*.yaml" --wrap-in config --to json --output-dir wrapped/

Install


pip install devbench
devbench cf config.yaml --wrap-in data --help

Works on Python 3.10–3.13. Fully offline — no telemetry, no external API calls. Supports 11 formats: JSON, YAML, TOML, XML, CSV, INI, .env, HCL, Java .properties, Apple plist, JSONC.
