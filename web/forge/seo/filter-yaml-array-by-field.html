Filter YAML Arrays by Field Value — devbench cf --select

Select array items matching a condition in any config format. One flag replaces yq's select() expressions, jq pipelines, and grep hacks.

The Problem: yq select() Is Hard to Remember

Filtering a Kubernetes container list with yq:

yq '.spec.containers[] | select(.name == "nginx")' deploy.yaml

That's three things to remember: the pipe, select(), and yq's expression syntax. Most DevOps engineers look it up every time.

The Solution: devbench cf --select

# Filter containers by name
devbench cf deploy.yaml --get spec.containers --select name=nginx

# Find running pods
devbench cf pods.yaml --select status=Running

# Exclude disabled services
devbench cf services.yaml --select enabled!=false

# Filter JSON arrays too
devbench cf instances.json --select role=primary --to yaml

--select FIELD=VALUE keeps list items where the field equals the value.
--select FIELD!=VALUE keeps items where the field does not match.

Kubernetes: Filter Containers by Name

Given deploy.yaml:

spec:
  containers:
  - name: nginx
    image: nginx:1.21
    ports:
    - containerPort: 80
  - name: redis
    image: redis:7
    ports:
    - containerPort: 6379
# Get the nginx container only
devbench cf deploy.yaml --get spec.containers --select name=nginx
# Output:
- image: nginx:1.21
  name: nginx
  ports:
  - containerPort: 80

Filter Pods by Status

devbench cf pods.yaml --select status=Running
devbench cf pods.yaml --select status!=Pending

Combine with --get for Nested Arrays

# Navigate to the array first with --get, then filter with --select
devbench cf config.yaml --get spec.services --select type=LoadBalancer

# Output as JSON
devbench cf k8s.yaml --get spec.containers --select name=api --to json

Type-Aware Value Matching

Values are automatically coerced before comparison:

  • --select replicas=3 matches integer 3, not string "3"
  • --select enabled=true matches boolean true
  • --select count=null matches Python None
# Filter services with replicas > 0 (equality on integer)
devbench cf services.yaml --select replicas=3

# Keep only enabled features
devbench cf features.yaml --select enabled=true

Exit Codes for Shell Scripting

devbench cf pods.yaml --select status=Running
# Exit 0 = matches found
# Exit 1 = no matches (grep semantics — safe for set -e)
# In a shell script:
if devbench cf pods.yaml --select status=Running > /dev/null; then
  echo "At least one pod is running"
else
  echo "No running pods found"
fi

Versus yq / jq / dasel

Get a Default When Key Is Missing

Combined with the new --default flag for safer shell scripts:

# Return default if path doesn't exist (never errors out in set -e scripts)
TIMEOUT=$(devbench cf config.yaml --get server.timeout --default 30)
RETRIES=$(devbench cf config.yaml --get retry.count --default 3)

# Equivalent to but simpler than:
TIMEOUT=$(devbench cf config.yaml --get server.timeout 2>/dev/null || echo 30)
