Minify JSON from the CLI — devbench cf --compact

Convert any config to compact single-line JSON in one command. Like jq -c but for YAML, TOML, INI, .env, XML, CSV, and 8 other formats.

Quick Start

# Install
pip install devbench

# Compact YAML → single-line JSON
devbench cf config.yaml --to json --compact --raw
# → {"server":{"host":"localhost","port":8080},"debug":true}

# TOML → compact JSON
devbench cf pyproject.toml --to json --compact --raw

# Already-JSON, just minify it
devbench cf big.json --to json --compact --raw

# Pipe into jq for further processing
devbench cf config.yaml --to json --compact --raw | jq .server

The Problem: Pretty JSON in Pipelines

Most config tools output pretty-printed JSON with indentation and newlines — fine for humans, awkward for pipelines:

# Problem: jq outputs pretty JSON by default
cat config.json | jq .
# {
#   "server": {
#     "host": "localhost",
#     "port": 8080
#   }
# }

# Problem: yq YAML→JSON also pretty-prints
yq -o json config.yaml

# You end up piping through jq -c:
cat config.yaml | yq -o json | jq -c .
# Multiple tools, multiple dependencies

The Solution: --compact flag

# One tool, one flag, one format
devbench cf config.yaml --to json --compact --raw
# → {"server":{"host":"localhost","port":8080},"debug":true}

# Works across ALL 11 formats — no jq needed
devbench cf app.toml    --to json --compact --raw
devbench cf vars.ini    --to json --compact --raw
devbench cf secrets.env --to json --compact --raw
devbench cf data.xml    --to json --compact --raw

Use Cases

API Payload Construction

# Build a compact JSON payload from YAML config
PAYLOAD=$(devbench cf deploy.yaml --pick spec.template.spec --to json --compact --raw)
curl -X POST https://api.example.com/deploy \
  -H "Content-Type: application/json" \
  -d "$PAYLOAD"

Log Ingestion

# Convert config snapshot to compact JSON for structured logging
devbench cf /etc/myapp/config.yaml --to json --compact --raw \
  | logger -t myapp-config

CI/CD Config Size Checks

# Compact JSON for size comparison
COMPACT=$(devbench cf config.yaml --to json --compact --raw)
BYTES=${#COMPACT}
echo "Config payload: ${BYTES} bytes"
[ "$BYTES" -lt 4096 ] || { echo "Config too large for env var"; exit 1; }

GitHub Actions

- name: Export config as compact JSON
  run: |
    echo "CONFIG=$(devbench cf config.yaml --to json --compact --raw)" >> $GITHUB_ENV

- name: Use compact config
  run: echo "Config is $CONFIG"

Combining with --pick

# Extract a specific section and compact it
devbench cf k8s-deploy.yaml --pick spec.template --to json --compact --raw

# Feed to another tool
devbench cf config.yaml --pick database --to json --compact --raw \
  | python3 -c "import json,sys; d=json.load(sys.stdin); print(d['host'])"

Flags

FlagDescription
--compact / -cCompact/minified JSON output — no spaces after : or ,
--raw / -rOutput just the config content, not the JSON envelope
--to jsonTarget format (json, yaml, toml, xml, csv, ini, env, hcl, properties, plist)
--from FORMATSource format (auto-detected by default)

Flag combinations

# --compact --raw: just the minified JSON string
devbench cf config.yaml --to json --compact --raw
# → {"key":"value"}

# --compact alone: compact JSON inside the tool envelope
devbench cf config.yaml --to json --compact
# → {"tool_name":"cf","output":"{\"key\":\"value\"}","success":true,...}

Compact JSON from TOML Not supported Not supported devbench cf f.toml --to json -c -r
Compact JSON from INI Not supported Not supported devbench cf f.ini --to json -c -r
Input from stdin ✅ ✅ ✅ cat f.yaml | devbench cf --to json -c -r

Install

pip install devbench
# or with all optional format support:
pip install devbench[all]

Works on macOS, Linux, and Windows. Python 3.10+.
