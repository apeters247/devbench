jq Alternative for CSV to JSON Conversion — Solving the 8-Year Gap (Issue #1650)


For eight years, users have been asking jq for one simple thing: convert a CSV string to JSON. jq issue #1650, opened in April 2018 with 37 comments and the label "feature request: jq function library", remains unresolved. The core problem: jq -r '@csv' only formats arrays as CSV — it cannot parse a CSV string into a structured format.


If you're searching for a jq alternative for CSV to JSON conversion, ConfigForge (devbench cf) fills this gap immediately. It converts CSV to JSON, CSV to YAML, CSV to TOML, and any of 9 supported formats — bidirectionally, 100% offline, with one command.


The Problem: jq @csv Is Write-Only


Here's what happens when you try to use jq to parse a CSV string today:


# jq CAN format a JSON array as CSV (this works)
$ echo '[{"name":"db","port":5432},{"name":"web","port":8080}]' | \
  jq -r '.[] | [.name, .port] | @csv'
"db",5432
"web",8080


# jq CANNOT parse a CSV string back into JSON (issue #1650)
$ echo '"db",5432
"web",8080' | jq -r '@csv'
# Error — jq has no CSV parser built in

This means any workflow that receives CSV data — database exports, spreadsheet downloads, API responses, log files — requires a secondary tool. Common workarounds include piping through mlr (Miller), csvkit, or writing custom Python scripts. Each adds a dependency and breaks the "one piping chain" promise.


The Solution: ConfigForge CSV Conversion


ConfigForge treats CSV as a first-class format alongside JSON, YAML, TOML, XML, INI, .env, HCL, and Java .properties. The same single command handles both directions:


# CSV to JSON — exactly what jq issue #1650 asks for
$ devbench cf convert servers.csv servers.json

# CSV to YAML (great for Ansible inventory)
$ devbench cf convert inventory.csv inventory.yaml

# CSV to TOML (app configs)
$ devbench cf convert spreadsheet.csv config.toml

# The reverse also works: JSON to CSV
$ devbench cf convert config.json export.csv

# Batch: convert all CSVs in a directory to JSON
$ devbench cf convert --batch --recursive "data/*.csv" --to json

Unlike jq's array-only @csv filter, ConfigForge handles:

  • Headers — First row becomes JSON keys automatically
  • Nested structures — Pipe-delimited headers (server.name) produce nested JSON
  • Quoted values — Commas inside quotes are handled correctly
  • Type inference — Numbers, booleans, and nulls are auto-detected
  • Multi-format output — CSV → any of 9 supported formats

Where jq Still Shines


To be clear: jq remains the gold standard for JSON querying and transformation. Its expression language for filtering, slicing, and reshaping JSON objects is unmatched. If your data starts and ends as JSON, jq is the right tool.


The gap is cross-format conversion — specifically taking CSV (or YAML, TOML, INI, or .env) and producing a structured output. That's where jq issue #1650 has sat unresolved for 8 years, and where ConfigForge provides an immediate answer.


Using Both Together: The Best of Both Worlds


The most productive setup uses jq for querying and ConfigForge for format conversion, chained in a single pipeline:


# jq reshapes JSON, ConfigForge converts to YAML
cat data.json | jq '.items[] | {id, name, status}' | devbench cf --to yaml

# ConfigForge converts CSV to JSON, jq filters it
devbench cf convert pipeline.csv -t json | jq '.[] | select(.status == "active")'

Together they cover the full spectrum of config and data processing — entirely offline, zero API calls, no web uploads.


Get Started


If you've been waiting 8 years for jq issue #1650 to be resolved, stop waiting:
