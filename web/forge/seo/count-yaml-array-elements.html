Count YAML / JSON Array Elements from the Command Line


Shell scripts that process config files often need a simple integer: how many containers are in this deployment? How many environment variables does the service define? How many services are declared? devbench cf --count outputs just that integer — no jq pipelines, no python -c, no grep | wc -l.


devbench cf deployment.yaml --count spec.template.spec.containers
# → 2

devbench cf config.yaml --count .
# → 5   (number of top-level keys)

Count Array Elements


# services.yaml
services:
  - name: api
    port: 8080
  - name: worker
    port: 9000
  - name: scheduler
    port: 9001


devbench cf services.yaml --count services
# → 3

Count Object Keys


Use . (dot) to count top-level keys, or any dot-path to count keys in a nested object:


# config.yaml
database:
  host: localhost
  port: 5432
  name: mydb
  user: admin
  pool_size: 10


devbench cf config.yaml --count database
# → 5

devbench cf config.yaml --count .
# → 1   (one top-level key: "database")

Shell Script Integration


Because --count outputs a plain integer, you can use it directly in arithmetic expressions and conditionals:


#!/bin/bash
# Fail if fewer than 2 containers are defined
n=$(devbench cf deployment.yaml --count spec.template.spec.containers)
if [ "$n" -lt 2 ]; then
  echo "ERROR: expected at least 2 containers, found $n" >&2
  exit 1
fi
echo "OK: $n containers"


# Count how many environment variables a Kubernetes pod declares
env_count=$(devbench cf pod.yaml --count spec.containers.0.env)
echo "Pod declares $env_count environment variables"

CI/CD: Assert Minimum Count


# .github/workflows/validate.yml
- name: Validate service count
  run: |
    pip install devbench
    n=$(devbench cf services.yaml --count services)
    [ "$n" -ge 1 ] || (echo "No services defined!" && exit 1)

- name: Validate replica count
  run: |
    replicas=$(devbench cf deployment.yaml --count spec.template.spec.containers)
    echo "Deployment has $replicas container(s)"

JSON Output for Structured Scripts


Add --raw to get a JSON object with path, count, and type fields:


devbench cf services.yaml --count services --raw


{"path": "services", "count": 3, "type": "array"}


devbench cf config.yaml --count database --raw


{"path": "database", "count": 5, "type": "object"}

Works With All 11 Formats


--count works on any format devbench can parse:


# Count items in a JSON array
devbench cf package.json --count dependencies

# Count TOML array of strings
devbench cf Cargo.toml --count dependencies

# Count CSV rows
devbench cf data.csv --count data

# Count XML child elements
devbench cf pom.xml --count dependencies
