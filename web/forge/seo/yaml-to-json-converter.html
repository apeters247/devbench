YAML to JSON Converter — Command-Line, Offline, YAML 1.2 Correct


Converting YAML to JSON from the command line sounds simple until you hit the edge cases: yes and no turning into true and false, Norwegian booleans (off, on), and multi-document YAML files that break JSON parsers. devbench cf handles all of these correctly with a single command.


# Basic YAML → JSON
devbench cf input.yaml -t json

# Write output to a file
devbench cf input.yaml -t json > output.json

# Pipe from stdin
cat input.yaml | devbench cf -f yaml -t json

# In-place conversion (replaces the file)
devbench cf input.yaml -t json --in-place

# Compact/minified JSON output
devbench cf input.yaml -t json --compact

The YAML Boolean Trap — Why Most Tools Get This Wrong


YAML 1.1 (the version most tools implement) treats these values as booleans:


enabled: yes      # YAML 1.1 → true
debug: no         # YAML 1.1 → false
flag: on          # YAML 1.1 → true
toggle: off       # YAML 1.1 → false

This causes silent data corruption in the "Norway Problem" — a YAML key named NO (Norway's ISO 3166-1 country code) becomes false. Any CI/CD pipeline or application reading the converted JSON would receive wrong values without any error.


devbench cf defaults to YAML 1.2, where only true and false are booleans. Everything else stays as a string:


# input.yaml
country: "NO"
enabled: yes
feature_flags:
  - on
  - off


devbench cf input.yaml -t json

{
  "country": "NO",
  "enabled": "yes",
  "feature_flags": ["on", "off"]
}

If you need YAML 1.1 behavior for legacy systems, use --yaml12 false:


# YAML 1.1 behavior (yes/no → true/false)
devbench cf input.yaml -t json --yaml12 false

Batch YAML to JSON Conversion


For converting entire directories of YAML files:


# Convert all YAML files in current directory
devbench cf "*.yaml" --batch -t json

# Recursive — all YAML files in a tree
devbench cf "**/*.yaml" --batch -t json

# With output directory (preserves directory structure)
devbench cf "configs/**/*.yaml" --batch -t json --output-dir ./json-configs/

# Stream mode for large numbers of files (constant memory usage)
devbench cf "**/*.yaml" --batch -t json --stream

Batch mode gives you per-file error reporting: if one file fails to parse, the rest continue converting. Errors go to stderr, output goes to files, so your pipeline stays clean.


Value Extraction Before Conversion


Use --get to extract a specific path from YAML and emit it as JSON:


# Extract a nested value as JSON
devbench cf kubernetes.yaml --get spec.containers -t json

# Extract and minify
devbench cf config.yaml --get database -t json --compact

# Extract with a fallback if the key is missing
devbench cf config.yaml --get database.timeout --default 30

# Filter a list by field value, then extract as JSON
devbench cf pods.yaml --select status.phase=Running --get metadata.name -t json

GitHub Actions — YAML Config to JSON in CI


- name: Convert config to JSON
  run: |
    pip install devbench
    devbench cf config.yaml -t json > config.json
    cat config.json | jq .database.host


- name: Validate YAML structure
  run: |
    pip install devbench
    devbench cf --check-env
    devbench cf config.yaml --validate
    devbench cf config.yaml --assert environment=production

Install


# Install devbench (includes cf command)
pip install devbench

# Verify
devbench cf --help
devbench cf --list-formats

Works on Python 3.10–3.13. Fully offline — no telemetry, no external API calls. The cf command supports 11 formats bidirectionally: JSON, YAML, TOML, XML, CSV, INI, .env, HCL, Java .properties, Apple plist, and JSONC.
