yq Alternative for Comment Preservation — Why ConfigForge Keeps Your Comments Intact


If you've ever run a YAML conversion with yq and noticed your comments vanished or got shuffled around, you're not alone. Two long-standing bugs — yq issue #462 (list indentation reformatting, open since June 2020) and yq issue #2516 (comments lost around merge-tags, open since November 2025) — have made yq users search for a yq alternative for comment preservation that treats documentation as first-class data, not noise to be discarded.


Enter ConfigForge (devbench cf): an offline, open-source CLI that preserves every comment across YAML, TOML, INI, and .env files — through conversions, round-trips, and batch processing. Below is a technical breakdown of the yq comment-handling bugs and how ConfigForge solves them.


What yq Gets Wrong with Comments


Issue #462: List Indentation Gets Reformatted (Open 6 Years)


When you use yq to process a YAML file, the list indentation silently changes:


# Input — minimal indentation, intentional
services:
- name: web
  port: 8080
- name: db
  port: 5432

# After yq processing — indentation inflated
services:
  - name: web
    port: 8080
  - name: db
    port: 5432

This may seem cosmetic, but it produces massive diffs in version control, breaks code review, and frustrates teams who rely on consistent formatting. The issue has been open since June 2020 with 26 👍 reactions and 18 comments — a clear sign of broad user demand that remains unresolved after six years.


Issue #2516: Comments Dropped Around Merge-Tags (Open, 0 Comments)


When using yq with merge-tags in an ireduce pipeline, comments preceding the << merge key are silently dropped:


# Input — with documentation
defaults: &defaults
  timeout: 30
  retries: 3

# Override the timeout for prod
prod:
  <<: *defaults
  timeout: 60

# After yq ireduce — comment lost!
prod:
  <<: *defaults
  timeout: 60

The comment explaining why the timeout was overridden is gone. For config files that serve as operational documentation, this is a knowledge loss that can lead to production incidents when future engineers can't understand the rationale.


How ConfigForge Preserves Comments


ConfigForge uses a fundamentally different approach: comments are extracted from the source format before parsing, attached to their corresponding data nodes, carried through the conversion pipeline, and re-inserted into the output. No node-level metadata is lost.


This works across all comment-supporting formats:


# YAML to JSON — comments naturally don't survive (JSON has no comments),
# but YAML to TOML does!
devbench cf convert docker-compose.yaml docker-compose.toml
# All # and // comments in docker-compose.yaml are preserved in
# the output docker-compose.toml

# Full round-trip: YAML → TOML → YAML — comments intact
devbench cf convert config.yaml config.toml
devbench cf convert config.toml config-roundtrip.yaml
# diff config.yaml config-roundtrip.yaml  # Only your changes, not comment loss

ConfigForge's comment preservation has been verified with dedicated test suites covering YAML, INI, and TOML round-trips — every test passes. There are no open issues for comment loss because the extraction-reinsertion pipeline handles merge-keys, anchors, inline comments, block comments, and file-level comments alike.

When Comment Preservation Matters Most


Infrastructure-as-Code repositories where YAML comments document the reasoning behind timeouts, feature flags, and dependency pins. Losing these comments erases institutional knowledge.


Migrating legacy configurations from INI or TOML to YAML — teams investing in comment documentation need to bring that context with them.


CI/CD pipelines that normalize configs across environments. Silent comment loss in automation is worse than manual: it affects every deployment without anyone noticing until the rationale is long gone.


Code review workflows where indentation reformatting (yq #462) pollutes diffs, making substantive changes harder to spot. Keeping indentation identical to the original file means reviews stay focused on what actually changed.


Get Started


If comment preservation is why you're looking for a yq alternative, ConfigForge is one command away:


pip install devbench-cf

# Convert YAML to TOML — comments preserved
devbench cf convert config.yaml config.toml

# Batch convert an entire directory — all comments survive
devbench cf convert --batch --recursive "**/*.yaml" --to toml
