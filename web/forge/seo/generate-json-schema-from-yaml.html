Generate JSON Schema from YAML, JSON, TOML — CLI One-Liner

--schema-gen 11 formats Draft 7 No yq/jq expression

DevBench's --schema-gen flag infers a complete JSON Schema Draft 7 document directly from any config file. It detects types, marks required fields, merges array item schemas, and handles nested objects — all automatically. No competitor (yq, dasel, jq) has this as a built-in flag.

Quickstart

# Install once
pip install devbench

# Generate schema from a YAML file
devbench cf config.yaml --schema-gen

# Output as YAML schema
devbench cf deploy.yaml --schema-gen --to yaml

# Save as a JSON Schema file for validation
devbench cf app.toml --schema-gen > app-schema.json

# Pipe the schema into jq for inspection
devbench cf config.json --schema-gen --raw | jq .properties

Example: Kubernetes Deployment → Schema

Input (deploy.yaml):

apiVersion: apps/v1
kind: Deployment
metadata:
  name: web-api
  namespace: production
spec:
  replicas: 3
  selector:
    matchLabels:
      app: web-api

Output (devbench cf deploy.yaml --schema-gen):

{
  "$schema": "http://json-schema.org/draft-07/schema#",
  "type": "object",
  "properties": {
    "apiVersion": {"type": "string"},
    "kind": {"type": "string"},
    "metadata": {
      "type": "object",
      "properties": {
        "name": {"type": "string"},
        "namespace": {"type": "string"}
      },
      "required": ["name", "namespace"]
    },
    "spec": {
      "type": "object",
      "properties": {
        "replicas": {"type": "integer"},
        "selector": {"type": "object", ...}
      },
      "required": ["replicas", "selector"]
    }
  },
  "required": ["apiVersion", "kind", "metadata", "spec"]
}

Array Item Schema Merging

When a list contains multiple objects, --schema-gen merges all item schemas. Fields present in every item are marked required; fields present in some are still included in properties.

# Input: list of Kubernetes containers
devbench cf containers.yaml --schema-gen

# Result: array with merged item schema
{
  "type": "array",
  "items": {
    "type": "object",
    "properties": {
      "name": {"type": "string"},
      "image": {"type": "string"},
      "ports": {"type": "array", ...}
    },
    "required": ["name", "image"]
  }
}

Real-World Use Cases

1. Document an undocumented API config

devbench cf prod-config.yaml --schema-gen > docs/config-schema.json
# Now you have a formal schema to share with your team

2. Bootstrap validation in CI/CD

# Generate schema once
devbench cf config.yaml --schema-gen > config-schema.json

# Validate new configs against it
devbench cf staging-config.yaml --schema config-schema.json
echo "Exit $?: config is valid"

3. OpenAPI schema generation

# TOML config → inline JSON Schema for OpenAPI spec
devbench cf settings.toml --schema-gen --raw | python3 -c "
import json, sys
schema = json.load(sys.stdin)
print(json.dumps({'requestBody': {'content': {'application/json': {'schema': schema}}}}, indent=2))
"

4. YAML schema for IDE autocompletion

# Generate YAML schema for VS Code / JetBrains YAML plugin
devbench cf config.yaml --schema-gen --to yaml > .vscode/config-schema.yaml

5. Validate multiple environments against a shared schema

# Generate schema from prod (the reference)
devbench cf configs/prod.yaml --schema-gen > shared-schema.json

# Validate all other environments
for env in staging dev test; do
  devbench cf configs/$env.yaml --schema shared-schema.json \
    && echo "$env: valid" || echo "$env: INVALID"
done

Cross-Format: TOML, INI, HCL, .env → JSON Schema

# TOML config → JSON Schema
devbench cf pyproject.toml --schema-gen

# HCL (Terraform) → JSON Schema
devbench cf main.tf --schema-gen --to yaml

# .env file → schema (useful for documenting required env vars)
devbench cf .env --schema-gen

# INI config → schema
devbench cf app.ini --schema-gen
