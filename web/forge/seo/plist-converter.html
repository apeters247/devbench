plist Converter CLI — plist ↔ YAML / JSON / TOML


macOS and iOS CI/CD pipelines regularly need to read and edit Info.plist files — bumping CFBundleVersion, toggling feature flags, setting environment-specific keys. The native tools are painful:


  • defaults write — can't handle nested dicts/arrays at all without rewriting entire keys
  • PlistBuddy — obscure syntax, macOS only, not available in Linux CI runners (GitHub Actions, GitLab CI)
  • plutil — converts between plist formats but doesn't work on Linux

ConfigForge (devbench cf) supports plist as a first-class format using Python's stdlib plistlib — no dependencies, no Xcode, works on macOS and Linux CI runners alike.


Common macOS CI/CD Use Cases


Bump CFBundleVersion in CI/CD


# Was: PlistBuddy -c "Set :CFBundleVersion 2.1.0" Info.plist
# Now:
devbench cf Info.plist --set CFBundleVersion 2.1.0 --in-place

# Verify
devbench cf Info.plist --get CFBundleVersion

Works on macOS and Linux CI runners (GitHub Actions ubuntu-latest, GitLab, Jenkins) — no PlistBuddy required.


Inspect plist as YAML for human review


# Was: plutil -convert json Info.plist -o - | python3 -m json.tool
# Now:
devbench cf Info.plist --to yaml

# Output:
# CFBundleIdentifier: com.example.MyApp
# CFBundleVersion: 1.0.0
# NSAppTransportSecurity:
#   NSAllowsArbitraryLoads: false
# UIRequiredDeviceCapabilities:
#   - armv7

Convert Info.plist to JSON for jq processing


# Extract a nested value with jq
devbench cf Info.plist --to json | jq '.NSAppTransportSecurity.NSAllowsArbitraryLoads'
# false

# Check minimum iOS version
devbench cf Info.plist --to json | jq -r '.MinimumOSVersion'
# 15.0

Set a nested key


# Enable arbitrary loads (for debug builds)
devbench cf Info.plist --set NSAppTransportSecurity.NSAllowsArbitraryLoads true --in-place

# Add a new key
devbench cf Info.plist --set CFBuildEnvironment staging --in-place

Round-trip: plist → YAML → plist


# Edit plist values in YAML (more readable), then convert back
devbench cf Info.plist --to yaml > info.yaml
# ... edit info.yaml in your editor ...
devbench cf info.yaml --to plist > Info.plist
