ENV to JSON Converter: .env Files to Structured JSON


Need to convert .env files to JSON for a monitoring dashboard? Centralizing deployment configuration? You need a ENV to JSON converter that handles batch conversion, preserves comments, and properly types values.


A Reddit user in r/DataHoarder: *"I have over 1000 .env files generated by our deployment system that I need to convert to JSON for a monitoring dashboard. File-by-file is not an option."*


ConfigForge handles glob patterns, batch conversion with progress, and produces clean JSON output. No uploads, no sign-ups, no limits.


Install



pip install devbench-cf

Or try the web demo: ConfigForge Web Demo


Example: Convert a Single .env File to JSON


Input: .env



# Application Settings
APP_NAME=my-web-app
APP_VERSION=1.5.2
NODE_ENV=production

# Server Configuration
HOST=0.0.0.0
PORT=8080
DEBUG=false
MAX_CONNECTIONS=1000

# Database
DATABASE_URL=postgresql://user:pass@localhost:5432/mydb
DATABASE_POOL_SIZE=25
DATABASE_SSL_ENABLED=true

# Features
ENABLE_CACHE=true
CACHE_TTL_SECONDS=3600
RATE_LIMIT_ENABLED=true
RATE_LIMIT_REQUESTS=100
ALLOWED_ORIGINS=["http://localhost:3000", "https://app.example.com"]

# Timestamps
DEPLOYED_AT=2024-06-15T10:30:00Z
BUILD_DATE=2024-06-14

Convert to JSON:



devbench cf .env --to json

Output: .env.json

Type Inference for ENV Files


ConfigForge detects real types in env values — not everything is a string:


ENV ValueJSON OutputCorrect Type |-----------|-------------|-------------| `DEBUG=false``"DEBUG": false`Boolean ✅ `PORT=8080``"PORT": 8080`Integer ✅ `MAX_CONNECTIONS=1000``"MAX_CONNECTIONS": 1000`Integer ✅ `RATE=3.14``"RATE": 3.14`Float ✅ `ORIGINS=["a","b"]``"ORIGINS": ["a","b"]`Array ✅ `NAME=hello``"NAME": "hello"`String ✅

Batch Convert 1000+ .env Files


This is where ConfigForge excels. Convert an entire directory of .env files in one command:



# Batch convert all .env files to JSON
devbench cf deployments/*.env --to json --out-dir ./json-output/

# With verbose progress
devbench cf deployments/*.env --to json --out-dir ./json-output/ --verbose

This processes every file, creating corresponding .json files in the output directory. Each file is typed, structured, and monitoring-ready.


ENV to JSON for Monitoring Dashboards


Convert deployment .env files into JSON for real-time dashboards:



# Deploy process
mkdir -p /var/monitor/env-json
devbench cf /var/deployments/*.env --to json --out-dir /var/monitor/env-json/

# Your monitoring tool reads the JSON files directly

Nested Key Support


For structured keys with dot-notation, ConfigForge can create nested JSON:



database.host=localhost
database.port=5432
database.name=mydb
logging.level=INFO
logging.file=/var/log/app.log

Command: devbench cf config.env --to json --expand-dots



{
  "database": {
    "host": "localhost",
    "port": 5432,
    "name": "mydb"
  },
  "logging": {
    "level": "INFO",
    "file": "/var/log/app.log"
  }
}
