Sort YAML / JSON List by Field Value — devbench cf --sort-by

devbench cf pods.yaml --sort-by metadata.name
Sort a list of objects by any field — ascending or descending. No yq expressions needed.

The Problem: Sorting Lists in yq / jq Is Error-Prone

Sorting a YAML list of objects by a field in yq or jq requires remembering expression syntax:

# yq: sort a list by a field value
yq '.items | sort_by(.metadata.name)' pods.yaml

# jq: sort list by a field
jq '.items | sort_by(.metadata.name)' pods.json

# yq: sort descending (verbose)
yq '.items | sort_by(.spec.replicas) | reverse' deploy.yaml

These expressions are format-specific, break on nested paths, and don't work on the top-level list without . wrapping.

The Solution: devbench cf --sort-by FIELD

# Sort Kubernetes pods by name
devbench cf pods.yaml --sort-by metadata.name

# Sort descending (highest replicas first)
devbench cf deploy.yaml --sort-by spec.replicas --sort-desc

# Sort a nested list after navigating with --get
devbench cf cluster.yaml --get spec.containers --sort-by name

# Sort and convert to JSON
devbench cf pods.yaml --sort-by metadata.name --to json

# Sort a CSV by a column value
devbench cf data.csv --sort-by price --sort-desc

Example: Sort Kubernetes Pods by Name

Input pods.yaml:

items:
  - metadata:
      name: web-c
      namespace: default
    spec:
      replicas: 2
  - metadata:
      name: api-a
      namespace: default
    spec:
      replicas: 5
  - metadata:
      name: db-b
      namespace: default
    spec:
      replicas: 1
devbench cf pods.yaml --get items --sort-by metadata.name

Output (sorted alphabetically by name):

- metadata:
    name: api-a
    namespace: default
  spec:
    replicas: 5
- metadata:
    name: db-b
    namespace: default
  spec:
    replicas: 1
- metadata:
    name: web-c
    namespace: default
  spec:
    replicas: 2

Sort Descending — Highest Replicas First

devbench cf pods.yaml --get items --sort-by spec.replicas --sort-desc

Output:

- metadata: {name: api-a, namespace: default}
  spec: {replicas: 5}
- metadata: {name: web-c, namespace: default}
  spec: {replicas: 2}
- metadata: {name: db-b, namespace: default}
  spec: {replicas: 1}

Compose with Other Flags

# Filter then sort: Running pods sorted by name
devbench cf pods.yaml --get items --select status.phase=Running --sort-by metadata.name

# Sort then extract: names of pods in replica order
devbench cf pods.yaml --get items --sort-by spec.replicas --sort-desc --each metadata.name

# Sort CSV rows by a numeric column
devbench cf report.csv --sort-by revenue --sort-desc --to json
