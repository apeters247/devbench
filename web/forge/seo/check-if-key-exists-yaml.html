Check If a Key Exists in YAML / JSON — devbench cf --has

devbench cf config.yaml --has database.host
Exit 0 if the path exists, exit 1 if missing. Shell-scriptable across 11 config formats.

The Problem: Testing Key Existence in Shell Scripts

Shell scripts often need to check whether a key exists before reading it. The yq/jq approaches are verbose and inconsistent:

# yq: check if key exists (verbose, fails for nested paths)
if yq '.database.host' config.yaml | grep -q 'null'; then
  echo "missing"
fi

# jq: check key exists
if [ "$(jq '.database.host // empty' config.json)" = "" ]; then
  echo "missing"
fi

Neither approach works across formats. ConfigForge's --has flag makes this trivial with a clean exit code:

The Solution: devbench cf --has PATH

# Check if a key exists — exit 0 = found, exit 1 = missing
devbench cf config.yaml --has database.host

# Use in a shell conditional
if devbench cf config.yaml --has database.host; then
  echo "host is configured"
fi

# Check nested path
devbench cf k8s.yaml --has spec.template.spec.containers

# Machine-readable output (prints true/false as JSON)
devbench cf config.yaml --has feature.flags.dark_mode --raw

Works Across All 11 Config Formats

# YAML
devbench cf app.yaml --has server.port

# TOML
devbench cf Cargo.toml --has dependencies.serde

# JSON
devbench cf package.json --has devDependencies.typescript

# .env
devbench cf .env --has DATABASE_URL

# INI
devbench cf config.ini --has database.host

# TOML → check then convert
if devbench cf pyproject.toml --has tool.ruff; then
  devbench cf pyproject.toml --get tool.ruff --to json
fi

Example: CI Guard Before Deployment

Input deploy.yaml:

app:
  name: my-service
  version: 1.2.0
database:
  host: db.internal
  port: 5432

Guard script:

#!/bin/bash
# Abort deploy if database credentials are not configured
for required_key in database.host database.port app.name; do
  if ! devbench cf deploy.yaml --has "$required_key"; then
    echo "ERROR: missing required config key: $required_key"
    exit 1
  fi
done
echo "Config OK — all required keys present"

Combine with --get and --default

# Check first, then safely read with a fallback
if devbench cf config.yaml --has cache.ttl; then
  TTL=$(devbench cf config.yaml --get cache.ttl)
else
  TTL=300
fi

# Or just use --default to handle missing keys gracefully
TTL=$(devbench cf config.yaml --get cache.ttl --default 300)
