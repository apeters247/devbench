Render Templates from Config Files — devbench cf --template

Use any config file as the data source for a template. Supports ${KEY} (Python string.Template) and {{ key }} (Jinja2). Works with YAML, JSON, TOML, and 8 more formats — no Helm, no Ansible required.

The Problem: Injecting Config Values Into Scripts

Injecting config values into a deploy script today requires jq + envsubst + multiple steps:

# Extract values first
export APP_PORT=$(yq '.server.port' config.yaml)
export APP_HOST=$(yq '.server.host' config.yaml)
export DB_HOST=$(yq '.database.host' config.yaml)

# Then substitute into template
envsubst < deploy.sh.tmpl > deploy.sh

That's two tools, three export lines per value, and YAML-only input.

The Solution: devbench cf --template

# Use YAML config to render a template
devbench cf config.yaml --template deploy.tmpl > deploy.sh

# Use TOML config to render an nginx config
devbench cf app.toml --template nginx.conf.tmpl > /etc/nginx/sites-available/app.conf

# Use JSON for a Kubernetes manifest
devbench cf values.json --template deployment.tmpl | kubectl apply -f -

Template Syntax: ${KEY} (Python string.Template)

Always available — no extra dependencies:

# deploy.tmpl
#!/bin/bash
HOST=${server_host}
PORT=${server_port}
DB=${database_host}

echo "Deploying to ${HOST}:${PORT}"
psql -h ${DB} -U ${database_user}
# config.yaml
server:
  host: prod.example.com
  port: 8080
database:
  host: db.internal
  user: appuser
devbench cf config.yaml --template deploy.tmpl
#!/bin/bash
HOST=prod.example.com
PORT=8080
DB=db.internal

echo "Deploying to prod.example.com:8080"
psql -h db.internal -U appuser

Dot-paths become underscores: server.host → ${server_host}. Uppercase variants also work: ${SERVER_HOST}.

Template Syntax: {{ key }} (Jinja2)

Install Jinja2 for richer templates:

pip install jinja2
# nginx.conf.tmpl
server {
    listen {{ server_port }};
    server_name {{ server_host }};

    location / {
        proxy_pass http://{{ backend_host }}:{{ backend_port }};
    }
}
devbench cf config.yaml --template nginx.conf.tmpl > /etc/nginx/sites-available/app.conf

GitHub Actions: Dynamic Workflow Variables

# workflow-config.yaml
deploy:
  environment: production
  region: us-east-1
  replicas: 3
  image_tag: v2.1.0
# deploy.tmpl
environment: ${deploy_environment}
region: ${deploy_region}
spec:
  replicas: ${deploy_replicas}
  template:
    spec:
      containers:
      - image: my-app:${deploy_image_tag}
devbench cf workflow-config.yaml --template deploy.tmpl | kubectl apply -f -

TOML Config → Shell Export Script

# pyproject.toml has your version
devbench cf pyproject.toml --template release.sh.tmpl > release.sh

Key Conventions

  • Nested keys flatten to underscores: database.host → ${database_host}
  • Both ${database_host} and ${DATABASE_HOST} work
  • If Jinja2 is installed, {{ database_host }} and {{ DATABASE_HOST }} work too
  • Falls back to Python string.Template if Jinja2 is not installed

Install

pip install devbench
# For Jinja2 templates:
pip install devbench jinja2

Works on Python 3.10–3.13. Supports 11 config formats: YAML, JSON, TOML, JSONC, XML, CSV, INI, .env, HCL, .properties, plist.
