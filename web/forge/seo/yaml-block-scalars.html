YAML Block Scalars: Fix Multiline String Corruption

ConfigForge   --block-scalars flag

PyYAML defaults to single-quoted style for multiline strings, producing unreadable output like 'step1\n step2\n '. The --block-scalars flag forces literal block scalar style (|) — clean, lossless, and exactly what DevOps engineers expect.

The Problem: Quoted vs Block Scalar

Without --block-scalars, multiline strings come out mangled:

# Input (JSON with multiline script)
{
  "install": "apt-get update\napt-get install -y curl\ncurl https://example.com/setup.sh | bash\n"
}

# Default PyYAML output — hard to read, trailing spaces corrupt
$ devbench cf install.json -t yaml
install: "apt-get update\napt-get install -y curl\ncurl https://example.com/setup.sh\
  | bash\n"

# With --block-scalars — clean, readable, lossless
$ devbench cf install.json -t yaml --block-scalars
install: |
  apt-get update
  apt-get install -y curl
  curl https://example.com/setup.sh | bash

Common Use Cases

Shell scripts in CI/CD config

# Kubernetes Job spec with install script
$ devbench cf job.json -t yaml --block-scalars
spec:
  template:
    spec:
      containers:
        - name: installer
          command:
            - /bin/sh
            - -c
          args:
            - |
              apt-get install -y curl
              curl -s https://setup.com | bash

PEM certificates in secrets

# TLS certificate stored in JSON → YAML with block scalars
$ devbench cf tls-secret.json -t yaml --block-scalars
tls.crt: |
  -----BEGIN CERTIFICATE-----
  MIIDXTCCAkWgAwIBAgIJAMB...
  -----END CERTIFICATE-----
tls.key: |
  -----BEGIN PRIVATE KEY-----
  MIIEvQIBADANBgkqhkiG9w0...
  -----END PRIVATE KEY-----

Ansible task descriptions

# Multi-line description in Ansible role metadata
$ devbench cf meta.json -t yaml --block-scalars
galaxy_info:
  description: |
    Installs and configures nginx as a reverse proxy.
    Supports SSL termination, rate limiting, and
    custom upstream pools.

Docker Compose entrypoints

$ devbench cf compose.json -t yaml --block-scalars
services:
  app:
    entrypoint:
      - /bin/sh
      - -c
    command: |
      echo "Starting server..."
      python manage.py migrate
      exec gunicorn -b 0.0.0.0:8080 app.wsgi

Cross-format: JSON → YAML with Block Scalars

# Any source format works
$ devbench cf github-actions.json -t yaml --block-scalars
$ devbench cf config.toml -t yaml --block-scalars
$ cat multiline.json | devbench cf - -f json -t yaml --block-scalars

Combine with Other Flags

# Sort keys + block scalars
$ devbench cf config.yaml --sort-keys --block-scalars

# Block scalars + in-place edit
$ devbench cf scripts.yaml -t yaml --block-scalars --in-place

# Batch conversion: all JSON files → clean YAML
$ devbench cf "configs/*.json" -t yaml --block-scalars --batch --output-dir yaml_out/

# Filter + block scalars
$ devbench cf job.yaml --get spec.template.spec --to yaml --block-scalars

API Usage

from core.configforge import serialize, parse_text

data = parse_text('{"script": "step1\\nstep2\\n"}', "json")
yaml_out = serialize(data, "yaml", block_scalars=True)
# script: |
#   step1
#   step2
