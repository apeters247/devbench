ConfigForge / Devbench — 11-format config converter

Convert YAML / TOML / JSON Config to Shell Variables

source <(devbench cf config.yaml --shell-export)
One command. Uppercase keys. Shell-quoted values. No jq, no awk, no manual escaping.

The Problem: Sourcing Config in Shell Scripts

DevOps pipelines often need config values in shell variables. The typical workflow is painful:

# ❌ Old way — fragile, format-specific, breaks on spaces
DB_HOST=$(grep 'host:' config.yaml | awk '{print $2}')
DB_PORT=$(grep 'port:' config.yaml | awk '{print $2}')
# Breaks on nested keys, multi-word values, TOML/JSON input

ConfigForge's --shell-export flag makes this trivial:

# ✅ New way — works on any format, handles quoting automatically
source <(devbench cf config.yaml --shell-export)
echo "$DB_HOST"   # localhost
echo "$DB_PORT"   # 5432

Quick Start

pip install devbench

# YAML config
cat config.yaml
# database:
#   host: localhost
#   port: 5432
#   name: mydb

devbench cf config.yaml --shell-export
# export DATABASE_HOST=localhost
# export DATABASE_PORT=5432
# export DATABASE_NAME=mydb

# Load into current shell
source <(devbench cf config.yaml --shell-export)
echo "$DATABASE_HOST"  # localhost

Real-World Use Cases

1. Docker Compose / entrypoint scripts

# docker-entrypoint.sh
set -euo pipefail
source <(devbench cf /etc/app/config.yaml --shell-export)

exec app --host "$DATABASE_HOST" --port "$DATABASE_PORT"

2. GitHub Actions — load config as environment variables

- name: Load config
  run: devbench cf config/prod.yaml --shell-export >> $GITHUB_ENV

3. CI/CD pipeline environment injection

# .gitlab-ci.yml
before_script:
  - pip install devbench
  - source <(devbench cf config/staging.yaml --shell-export)
  - echo "Deploying to $DATABASE_HOST"

4. Local dev environment setup from config

# In your .bashrc or project .envrc (direnv)
source <(devbench cf .devconfig.toml --shell-export)
# Now $APP_PORT, $DB_HOST, $REDIS_URL are all set

5. TOML config to shell (yq can't write TOML, and can't do this at all)

# pyproject.toml style
cat settings.toml
# [database]
# host = "prod.db.internal"
# port = 5432
# [cache]
# ttl = 300

devbench cf settings.toml --shell-export
# export DATABASE_HOST='prod.db.internal'
# export DATABASE_PORT=5432
# export CACHE_TTL=300

6. Kubernetes ConfigMap to shell variables

# Extract env-relevant keys from a k8s ConfigMap
devbench cf configmap.yaml --grep 'app\.' --shell-export

Flags and Options

# Machine-readable JSON (for scripting)
devbench cf config.yaml --shell-export --raw
# {
#   "exports": [
#     {"key": "DATABASE_HOST", "value": "localhost"},
#     {"key": "DATABASE_PORT", "value": "5432"}
#   ]
# }

# Only export keys matching a pattern
devbench cf config.yaml --grep 'database' --shell-export

# TOML / JSON / INI — format is auto-detected
devbench cf config.toml --shell-export
devbench cf config.json --shell-export
devbench cf config.ini  --shell-export

# Pipe from another command
cat secrets.yaml | devbench cf --shell-export

Comparison: ConfigForge vs Manual Shell Approaches

Task Old (grep/awk/sed) ConfigForge
Export flat YAML keys grep -v '^#' | awk -F': ' '{print "export "$1"="$2}' devbench cf cfg.yaml --shell-export
Export nested YAML keys ❌ Requires Python/jq + post-processing devbench cf cfg.yaml --shell-export
Export TOML keys ❌ No standard tool for TOML→shell export devbench cf cfg.toml --shell-export
Handle values with spaces ❌ Usually breaks quoting ✅ shlex.quote handles all edge cases
Uppercase key conversion tr '[:lower:]' '[:upper:]' + dot handling ✅ Automatic (dots, dashes → underscores)
Boolean values ❌ Exported as YAML true or Python True ✅ Always exported as true / false
JSON output for automation ❌ Not available ✅ --raw emits {exports: [{key, value}]}
