Extract a Field from Every Item in a YAML List — devbench cf --each

Pull a single key from every element of a config array in any format. One flag replaces jq array iteration expressions and yq map operations.

The Problem: jq Array Iteration Is Verbose

Extracting all container names with jq:

kubectl get pods -o json | jq '[.items[].spec.containers[].name]'

With yq:

yq '.spec.containers[].name' deploy.yaml

Both work, but you need to remember the syntax for each tool, and neither handles TOML, INI, or .env files.

The Solution: devbench cf --each

# Get all container names from a deployment
devbench cf deploy.yaml --get spec.containers --each name

# Output (JSON by default):
["app", "sidecar", "proxy"]

--each KEY extracts KEY from every element of a list. Items missing the key are silently omitted. Output is a list in any format you choose.

Works on Every Format

Unlike jq (JSON only) or yq (YAML only), --each works on all 12 supported formats:

# From TOML
devbench cf services.toml --each name --to json

# From JSON
devbench cf package.json --get dependencies --each name --to yaml

# From CSV (list of records)
devbench cf hosts.csv --each hostname --to json

Chain with --select to Filter First

The real power comes from chaining --select and --each:

# Get names of only Running pods
devbench cf pods.yaml --select status=Running --each metadata.name

# Get images of containers matching a name pattern (with --get to navigate first)
devbench cf deploy.yaml --get spec.containers --select name=nginx --each image

This replaces:

# jq equivalent (much harder to remember)
jq '[.[] | select(.status == "Running") | .metadata.name]' pods.json

# yq equivalent
yq '.[] | select(.status == "Running") | .metadata.name' pods.yaml

Dot-Notation for Nested Fields

KEY supports dot-notation for nested paths:

# Extract nested ports from a service list
devbench cf services.yaml --each spec.ports.0.port

# Extract nested metadata names from a list of k8s resources
devbench cf resources.yaml --each metadata.name

Real-World Examples

Kubernetes: List all container images in a deployment

devbench cf deploy.yaml --get spec.template.spec.containers --each image

Docker Compose: Get all service names

# Compose files use a map, not a list — navigate first
devbench cf docker-compose.yaml --keys

GitHub Actions: Extract all step names from a workflow

devbench cf .github/workflows/ci.yml --get jobs.build.steps --each name

Terraform: Get all resource names from a plan (JSON)

terraform show -json | devbench cf - --get resource_changes --each address

Script: Get all service names and iterate in bash

for svc in $(devbench cf services.yaml --each name --to json | jq -r '.[]'); do
  echo "Deploying $svc"
done
