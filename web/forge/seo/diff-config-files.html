Diff Config Files Across Formats from the CLI

ConfigForge 1.0 11 formats structural diff

Text-based diff compares lines. Config files aren't documents — they're structured data. A YAML file with different indentation, a JSON file with reordered keys, or two files in entirely different formats (YAML vs TOML) will show hundreds of spurious text differences even when the data is identical.

ConfigForge --diff parses both files first, normalizes them to dot-notation key paths, then compares the data structures. You see only real changes — added keys, removed keys, and value changes — not formatting noise.

Quick Start

# Install
pip install devbench

# Compare two YAML files structurally
devbench cf deploy-v1.yaml --diff deploy-v2.yaml

# Compare a YAML file against its JSON equivalent (cross-format!)
devbench cf config.yaml --diff config.json

# CI/CD: exit 0 if identical, exit 1 if different
devbench cf staging.yaml --diff production.yaml || echo "Environments differ"

# Machine-readable output for scripting
devbench cf base.yaml --diff override.yaml --raw

Why diff Falls Short for Config Files

Standard Unix diff compares text line-by-line. For config files this causes three problems:

  • Formatting differences masquerade as data differences — a YAML file reindented by your editor shows 50 changed lines even if no value changed.
  • Key reordering is invisible — moving a key block to a different position in a JSON file makes diff report every line changed, even though the data is identical.
  • Cross-format comparison is impossible — diff staging.yaml production.json always shows 100% different, even if the data matches perfectly.

ConfigForge solves all three by parsing first, then comparing data structures.

Real-World Use Cases

1. Kubernetes Staging vs Production Audit

# Find what's different between staging and production manifests
devbench cf k8s/staging/deploy.yaml --diff k8s/production/deploy.yaml

# Output:
# ~ spec.replicas: 2 -> 5
# ~ spec.template.spec.containers[0].resources.limits.memory: 512Mi -> 2Gi
# ~ spec.template.spec.containers[0].image: nginx:1.24 -> nginx:1.25

# Fail CI if environments diverge unexpectedly
devbench cf staging.yaml --diff production.yaml --raw \
  | jq '.identical' | grep -q true || exit 1

2. Cross-Format Infrastructure Comparison

# Your team uses YAML but a vendor sends JSON — compare directly
devbench cf my-config.yaml --diff vendor-config.json

# Compare Terraform HCL outputs vs expected JSON
devbench cf expected.json --diff actual.hcl

# No other tool can do this — yq is YAML-only, jq is JSON-only

3. PR / Code Review for Config Changes

# Compare HEAD vs main branch configs in CI
git show main:config.yaml > /tmp/base.yaml
devbench cf /tmp/base.yaml --diff config.yaml

# Output only the changed keys (not the identical ones)
devbench cf old.yaml --diff new.yaml --raw | jq '.changed'

4. Docker Compose Version Comparison

# Compare docker-compose files across environments
devbench cf docker-compose.dev.yaml --diff docker-compose.prod.yaml

# Typical output:
# ~ services.app.environment.DEBUG: true -> false
# ~ services.app.replicas: 1 -> 3
# + services.nginx (new service in prod)
# - services.mailhog (dev-only service removed)

5. Ansible Playbook Drift Detection

# Detect configuration drift in Ansible vars files
for env in dev staging prod; do
  echo "=== $env vs baseline ==="
  devbench cf vars/baseline.yaml --diff vars/$env.yaml
done

Output Formats

Human-readable (default)

$ devbench cf deploy-v1.yaml --diff deploy-v2.yaml
- metadata.labels.version: 'v1'          # removed
+ spec.template.spec.initContainers      # added
~ spec.replicas: 2 -> 5                  # changed
~ spec.template.spec.containers[0].image: nginx:1.24 -> nginx:1.25
SymbolMeaning
+Key/section added in the second file
-Key/section removed (present in first, absent in second)
~Value changed — shows old -> new

JSON output (--raw)

$ devbench cf v1.yaml --diff v2.yaml --raw
{
  "identical": false,
  "formats": ["yaml", "yaml"],
  "added": ["spec.template.spec.initContainers"],
  "removed": ["metadata.labels.version"],
  "changed": {
    "spec.replicas": {"from": 2, "to": 5},
    "spec.template.spec.containers[0].image": {
      "from": "nginx:1.24",
      "to": "nginx:1.25"
    }
  }
}
CI/CD tip: --diff exits 0 when files are identical, 1 when they differ. Use this as a gate: devbench cf expected.yaml --diff actual.yaml || { echo "Config drift detected"; exit 1; }

Flags

FlagDescription
--diff FILEFile to compare against (any supported format)
--rawJSON output — identical, formats, added, removed, changed
--from FORMATForce first file's format (default: auto-detect)

Exit codes: 0 = identical, 1 = differences found — same semantics as Unix diff.

ConfigForge is the only CLI tool with native cross-format structural diff. yq, jq, and dasel all lack a --diff mode. The common workaround — convert both to JSON then pipe to jq — loses type information and fails on TOML, INI, and ENV formats.

Shell Script Examples

# Gate: fail CI if staging and production configs diverge
devbench cf staging/config.yaml --diff production/config.yaml
if [ $? -ne 0 ]; then
  echo "ERROR: staging and production configs differ — review before deploy"
  exit 1
fi

# Count how many keys changed between versions
CHANGES=$(devbench cf v1.yaml --diff v2.yaml --raw \
  | jq '.changed | length')
echo "$CHANGES keys changed"

# Alert if a specific critical key changed
devbench cf old.yaml --diff new.yaml --raw \
  | jq '.changed | has("database.primary.host")' | grep -q true \
  && echo "WARNING: database host changed — verify connection strings"

# Generate a changelog from config differences
devbench cf config-1.0.yaml --diff config-1.1.yaml --raw \
  | jq -r '.changed | to_entries[] | "  ~ \(.key): \(.value.from) -> \(.value.to)"'

GitHub Actions Integration

name: Config Drift Check
on: [pull_request]

jobs:
  drift:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with: {fetch-depth: 2}

      - run: pip install devbench

      - name: Check config drift
        run: |
          git show HEAD~1:config/production.yaml > /tmp/prev.yaml
          devbench cf /tmp/prev.yaml --diff config/production.yaml --raw \
            | tee diff-report.json
          # Fail if critical keys changed
          jq -e '.identical or (.changed | keys | map(select(startswith("database."))) | length == 0)' \
            diff-report.json
