Grep YAML, JSON & TOML Config Files from the CLI

ConfigForge 1.0 11 formats regex search

DevOps engineers spend hours grep-ing YAML files — but standard grep doesn't understand structure. It returns raw YAML lines, not key paths. If the value you're looking for appears in a multi-line block scalar or is deeply nested, you miss it.

ConfigForge --grep parses the config first, flattens it to dot-notation paths, then applies your regex. You get structured results: spec.template.spec.containers[0].image: 'nginx:1.21', not a raw line dump.

Quick Start

# Install
pip install devbench

# Search a Kubernetes manifest for any nginx reference
devbench cf deploy.yaml --grep 'nginx'

# Find all image references (regex)
devbench cf deploy.yaml --grep 'image'

# Security audit: find any password/secret key in all YAML files
devbench cf '*.yaml' --batch --grep 'password|secret|token'

# Structured JSON output for scripting
devbench cf config.yaml --grep 'host' --raw

Why grep -r Falls Short for Config Files

Standard grep treats config files as plain text. Problems:

  • No path context — you see image: nginx but not which container
  • Misses block scalars — multi-line YAML values split across lines
  • Cross-format blind — can't grep a mix of YAML, JSON, and TOML in one pass
  • No value types — can't distinguish a value of 80 from a comment containing 80

ConfigForge parses first, then matches. You always get structured key-path results.

Real-World Use Cases

1. Kubernetes Security Audit

# Find any hardcoded secrets in all manifests
devbench cf 'k8s/*.yaml' --batch --grep 'password|token|secret|key' --raw

# Find containers using the latest tag (unstable)
devbench cf deploy.yaml --grep ':latest'

# Find all service account references
devbench cf '**/*.yaml' --batch --grep 'serviceAccount' --recursive

2. Docker Compose Dependency Discovery

# Find all image references across services
devbench cf docker-compose.yaml --grep 'image:'

# Find all port mappings
devbench cf docker-compose.yaml --grep '^ports\.'

# Find which services have health checks
devbench cf docker-compose.yaml --grep 'healthcheck'

3. CI/CD Pipeline Config Audit

# Search GitHub Actions workflows for deprecated actions
devbench cf '.github/workflows/*.yaml' --batch --grep 'actions/checkout@v1' --recursive

# Find all environment variable references
devbench cf .github/workflows/deploy.yaml --grep '\$\{\{.*secrets\.'

# Check Terraform variables for defaults
devbench cf '*.tfvars' --batch --grep 'region'

4. Cross-Format Grep

# Grep across mixed YAML and JSON config dirs
devbench cf 'config/**' --batch --grep 'localhost' --recursive

# Find debug flags across all formats
devbench cf '*.{yaml,json,toml,ini}' --batch --grep 'debug.*true'

Output Formats

Human-readable (default)

$ devbench cf deploy.yaml --grep 'image'
spec.template.spec.containers[0].image: 'nginx:1.21'
spec.template.spec.containers[1].image: 'redis:7'
spec.initContainers[0].image: 'busybox:latest'

JSON output (--raw)

$ devbench cf deploy.yaml --grep 'image' --raw
{
  "file": "deploy.yaml",
  "pattern": "image",
  "matches": [
    {"path": "spec.template.spec.containers[0].image", "value": "nginx:1.21"},
    {"path": "spec.template.spec.containers[1].image", "value": "redis:7"},
    {"path": "spec.initContainers[0].image", "value": "busybox:latest"}
  ],
  "count": 3
}

Batch output (--batch)

$ devbench cf '*.yaml' --batch --grep 'password'
--- db.yaml (2 matches) ---
  database.primary.password: 'hunter2'
  database.replica.password: 'hunter2'
--- app.yaml (0 matches) ---

Flags

FlagDescription
--grep PATTERNRegex pattern to match against key paths and values
--grep-case-sensitiveCase-sensitive match (default: case-insensitive)
--rawJSON output — file, pattern, matches array, count
--batchTreat text argument as a glob; grep each matched file
--recursive / -REnable ** glob recursion in --batch
--fromForce input format (default: auto-detect)

Exit codes: 0 = matches found, 1 = no matches — same as GNU grep, scripts can use if devbench cf config.yaml --grep secret; then ...

grep -r Plain text only ✗ raw line context ✓ ✓ ✗
yq (YAML only) YAML / JSON only ✓ with filter expr ✗ no native batch ✗ no native grep ✓
jq (JSON only) JSON only ✓ with filter expr ✗ no native batch ✓ test() ✓
dasel YAML/JSON/TOML/XML/CSV ✓ with selector ✗ ✗ no grep mode ✓
No competitor has a native --grep equivalent. yq and jq require you to know the exact path first, or write complex .. | select(test("pattern")) expressions. ConfigForge just takes a pattern.

Shell Script Examples

# Count secrets in a directory (CI gate)
SECRET_COUNT=$(devbench cf 'k8s/*.yaml' --batch --grep 'password' --raw \
  | python3 -c 'import json,sys; data=json.load(sys.stdin); print(sum(f["count"] for f in data))')

if [ "$SECRET_COUNT" -gt 0 ]; then
  echo "ERROR: $SECRET_COUNT hardcoded secrets found"
  exit 1
fi

# Find all image tags and check for :latest
devbench cf deploy.yaml --grep ':latest' && {
  echo "Warning: :latest tags found — pin versions for reproducibility"
}

# Extract all unique hostnames from config files
devbench cf config.yaml --grep 'host' --raw \
  | jq -r '.matches[].value' \
  | sort -u
