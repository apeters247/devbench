yq Can't Write TOML — devbench cf Can

mikefarah/yq issue #1364 — "TOML support" — has been open since April 2020. yq can read TOML, but it cannot write it. If you need YAML→TOML, JSON→TOML, or INI→TOML from the command line, devbench cf is the only general-purpose tool that ships native TOML output.

The Problem

# yq: reads TOML fine, but can't write it
yq -o=toml input.yaml   # Error: unsupported output format 'toml'

# kislyuk/yq: JSON-only output (wraps jq)
cat input.yaml | yq .   # YAML→JSON only; no TOML path

# dasel: silently drops comments, no TOML write either
dasel -f yaml -o toml input.yaml   # Error or empty output

Users have been workarounds-posting on Reddit and Stack Overflow since 2020: use Python inline, use python -c "import toml, yaml; ...", use a Dockerfile with multiple deps. There's no clean one-liner.

The Fix: devbench cf -t toml

pip install devbench

# YAML → TOML
devbench cf config.yaml -t toml

# JSON → TOML
devbench cf package.json -t toml

# INI → TOML
devbench cf setup.cfg -t toml

# In-place conversion (rewrites the file)
devbench cf pyproject.yaml -t toml --in-place

# Stdin → TOML (pipe-friendly)
cat config.yaml | devbench cf -f yaml -t toml

Real-World Examples

pyproject.toml Migration

Many projects maintain a setup.cfg or setup.yaml and want to migrate to PEP 517/518 pyproject.toml:

# Before: setup.cfg (INI format)
cat setup.cfg
# [metadata]
# name = mypackage
# version = 1.0.0
# [options]
# install_requires = requests>=2.28

devbench cf setup.cfg -t toml
# [metadata]
# name = "mypackage"
# version = "1.0.0"
#
# [options]
# install_requires = ["requests>=2.28"]

Cargo.toml — Check if Keys Exist

# Read values from TOML (yq can do this)
devbench cf Cargo.toml --get package.version

# Write TOML from another format (yq cannot do this)
devbench cf deps.yaml -t toml >> Cargo.toml

Kubernetes ConfigMap to TOML App Config

# Export ConfigMap data section as TOML for an app that reads TOML natively
kubectl get configmap app-config -o json \
  | devbench cf -f json --get data -t toml

Helm values.yaml → TOML for Rust/Go services

# Helm renders values.yaml; your service reads TOML. Convert at deploy time:
helm show values myrepo/mychart | devbench cf -f yaml -t toml > config.toml

Why Not Use Python Inline?

# The common workaround (fragile, not pipeable, requires both pyyaml + toml)
python3 -c "import sys, yaml, toml; print(toml.dumps(yaml.safe_load(sys.stdin)))"

# Problems:
# - Requires pyyaml AND toml/tomli_w to be installed
# - No error handling
# - No --get, --select, --merge composability
# - Can't process multiple files
# - Breaks on YAML anchors, multiline strings, dates

devbench cf is a single pure-Python dependency that handles all 11 formats without needing jq, yq, or format-specific packages separately.

Install

pip install devbench
# or: pip install devbench[all]  # includes all optional format deps

devbench cf --help
devbench cf --list-formats   # shows all 11 supported formats

Works on Python 3.10–3.13, macOS, Linux, and Windows. No Homebrew, no binary download.
