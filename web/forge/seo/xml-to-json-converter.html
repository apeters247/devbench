XML to JSON Converter — Command-Line, Offline, Attribute-Aware


Converting XML to JSON is deceptively complex: attributes, namespaces, CDATA sections, and text nodes all need conventions. devbench cf uses a consistent attribute encoding (prefixed with @) so the output is predictable and round-trip-safe.


# Convert XML to JSON
devbench cf input.xml -t json

# Convert XML to YAML
devbench cf input.xml -t yaml

# Pipe from stdin
cat input.xml | devbench cf -f xml -t json

# Compact/minified JSON
devbench cf input.xml -t json --compact

How Attributes Are Handled


XML attributes and child elements coexist in the same namespace. devbench encodes attributes with an @ prefix to distinguish them from child elements:


<?xml version="1.0"?>
<server host="db.example.com" port="5432">
  <database>myapp_prod</database>
  <ssl enabled="true"/>
</server>


devbench cf config.xml -t json

{
  "server": {
    "@host": "db.example.com",
    "@port": "5432",
    "database": "myapp_prod",
    "ssl": {
      "@enabled": "true"
    }
  }
}

Attribute values are always strings in XML, so they remain strings in the JSON output. If you need integer coercion, pipe the result through jq or use devbench cf again with --set.


XML to YAML


Converting to YAML preserves the same attribute encoding and is often more readable for config files:


devbench cf config.xml -t yaml

server:
  '@host': db.example.com
  '@port': '5432'
  database: myapp_prod
  ssl:
    '@enabled': 'true'

Batch XML to JSON Conversion


For converting entire directories of XML files:


# Convert all XML files in current directory
devbench cf "*.xml" --batch -t json

# Recursive
devbench cf "**/*.xml" --batch -t json

# With output directory (preserves tree structure)
devbench cf "data/**/*.xml" --batch -t json --output-dir ./json-out/

# Stream mode — constant memory for thousands of files
devbench cf "**/*.xml" --batch -t json --stream

Batch mode gives you per-file error reporting. Malformed XML files log an error to stderr and are skipped; the rest continue converting.


Extract Values from XML


Use --get to extract a specific path after parsing:


# Get a specific element value
devbench cf config.xml --get server.database

# Get an attribute
devbench cf config.xml --get 'server.@host'

# Get with fallback
devbench cf config.xml --get server.timeout --default 30

# Convert and extract in one step
devbench cf config.xml --get server -t json

Maven pom.xml — Extract Dependencies


# Extract all dependencies from a Maven pom.xml
devbench cf pom.xml --get project.dependencies.dependency -t json

# Count dependencies
devbench cf pom.xml --get project.dependencies.dependency --count

# Find dependencies matching groupId
devbench cf pom.xml --get project.dependencies.dependency \
  --select groupId=org.springframework -t yaml

Android / iOS plist and Manifest Files


# Convert Android AndroidManifest.xml to JSON for CI processing
devbench cf AndroidManifest.xml -t json --compact

# Convert Apple plist XML to YAML
devbench cf Info.plist -t yaml

# Extract version string from iOS plist
devbench cf Info.plist --get 'plist.dict.CFBundleShortVersionString'

Install


pip install devbench

# Verify XML support
devbench cf --list-formats

Works on Python 3.10–3.13. Fully offline — no telemetry, no external API calls. Supports 11 formats bidirectionally: JSON, YAML, TOML, XML, CSV, INI, .env, HCL, Java .properties, Apple plist, JSONC.
