Troubleshooting Guide

Devbench doesn't appear in the menubar

  1. Ensure you're running macOS 13+.
  2. Check that Devbench is in your Applications folder, not running from a disk image.
  3. Try launching it from Finder or the Applications folder, not Spotlight.
  4. If you see a security warning, go to System Settings → Privacy & Security and click Open Anyway.
  5. Restart your Mac and try again.

Clipboard auto-detect isn't working

  1. Make sure you've granted Devbench Accessibility permissions in System Settings → Privacy & Security → Accessibility.
  2. Try copying data after launching Devbench (not before).
  3. Restart Devbench and try again.

The app says "Damaged" or won't open

  1. Open Terminal and run: xattr -dr com.apple.quarantine /Applications/Devbench.app
  2. Then try launching Devbench again.
  3. If it still doesn't open, re-download the app and repeat.

Keyboard shortcuts aren't working

  1. Check that Devbench is actually running (look for the icon in your menubar).
  2. Verify no other app is using the same shortcut (System Settings → Keyboard → Keyboard Shortcuts).
  3. You can customize shortcuts in Devbench's Preferences panel.
