## Summary

The comment-preservation feature already existed in `core/configforge.py` (extract → carry through JSON intermediate via `convert(..., _carry_comments=...)` → re-insert, exposed ergonomically through `round_trip()`). But probing it surfaced two genuine round-trip bugs where comments were silently destroyed — exactly the #1 complaint. I fixed both via TDD.

**Bugs found & fixed** (`core/configforge.py`):

1. **List-item inline comments were dropped.** `- apple  # a tasty fruit` was extracted with `key=None`, classified as "standalone", and standalone *inline* comments were never re-inserted. Now list-item comments are anchored to their item value and re-attached to the matching `-` line on the return leg.

2. **A comment after a value containing a `#` was lost.** The extractor only inspected the *first* `#` on a line and gave up if it sat inside quotes — so `color: "#ffffff"  # background color` lost its real trailing comment. The re-inserter's `"#" not in stripped` guard mis-fired for the same reason. Both now use a new `_yaml_find_comment()` helper that applies the real YAML rule: a `#` opens a comment only when it's outside quotes *and* at line-start or preceded by whitespace. (This also stops `url: http://x#frag` being mistaken for a comment.)
