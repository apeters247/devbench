# Convert Kubernetes YAML to JSON: A Complete Guide for K8s Configs

If you've searched for **how to convert Kubernetes YAML to JSON**, you already know the challenge: K8s manifests use multi-document YAML with `---` separators, nested resource specs, and inline anchors. Most YAML-to-JSON converters either crash on multi-document files or silently process only the first document.

This guide covers every method — from `kubectl` to dedicated CLI tools — and shows when each one works best.

## Method 1: Using kubectl (When It Works)

When you have a running cluster, `kubectl` can output JSON natively:

```bash
kubectl get deployment myapp -o json
```

Or get the raw manifest without cluster defaults:

```bash
kubectl get deployment myapp -o json --dry-run=client
```

**Limitations of kubectl:**
- Requires a live cluster connection — no offline use
- Only works for Kubernetes kinds — can't convert arbitrary YAML
- Injects cluster defaults (status fields, defaulted values)
- No batch mode for directory conversion

## Method 2: Using ConfigForge CLI for Offline Conversion

For offline, private conversion of any YAML file — not just K8s — ConfigForge (`devbench cf`) gives you full control:

```bash
devbench cf --to json < deployment.yaml
```

This works without a cluster, without internet, and handles any valid YAML — not just Kubernetes kinds.

## Handling Multi-Document YAML (The Kubernetes Challenge)

A single Kubernetes manifest file often contains multiple resources separated by `---`:

```yaml
apiVersion: v1
kind: Namespace
metadata:
  name: production
---
apiVersion: apps/v1
kind: Deployment
metadata:
  name: webapp
---
apiVersion: v1
kind: Service
metadata:
  name: webapp-svc
```

Most converters crash here or process only the first `---` document. ConfigForge handles multi-document YAML natively, converting each document separately:

```bash
devbench cf --to json < multi-resource.yaml
```

You get a valid JSON representation of all resources — namespace, deployment, and service — preserving the document structure.

## Batch Converting an Entire Manifests Directory

When you have a `manifests/` directory with 100+ YAML files that need conversion:

```bash
devbench cf convert "manifests/*.yaml" --to json --batch
```

One command converts every `.yaml` file to `.json` with per-file progress reporting.

## Converting JSON Back to YAML for Editing

After modifying K8s configs in JSON (e.g., a CI pipeline processes JSON), convert back to YAML:

```bash
devbench cf --to yaml < modified.json
```

## Converting Helm values.yaml to JSON

Helm's `values.yaml` is pure YAML — `kubectl` can't help here since there's no `kind`:

```bash
devbench cf --to json < values.yaml
```

Useful when generating environment-specific configs from a values template.

## Security: Why You Shouldn't Upload K8s Configs Online

Kubernetes manifests contain sensitive information:
- Service names and internal DNS patterns
- Image repository URLs and tags
- Base64-encoded secrets (which are not encrypted)
- Persistent volume claims exposing storage topology
- ConfigMap contents with application settings

**Never paste K8s manifests into an online YAML-to-JSON converter.** ConfigForge keeps all data on your machine — zero API calls, zero uploads, zero logs on third-party servers.
