# How to Convert JSON to YAML from the Command Line — A Complete Guide

Stop pasting JSON into browser converters. This guide shows you how to convert JSON to YAML from the command line using **ConfigForge** (`devbench cf`) — a free, offline CLI that supports 9 config formats. No data leaves your machine, no accounts required, no rate limits.

## Why Use a CLI for JSON to YAML Conversion?

Online JSON to YAML converters have three problems:
- **Privacy risk** — You paste production configs into a third-party server
- **Format limits** — Most sites only convert one format pair (JSON ↔ YAML), not TOML, XML, CSV, or others
- **No batch mode** — One file at a time, manually

A local CLI solves all three. Let's get started.

## Basic JSON to YAML Conversion

The simplest form: name your source and target files, and ConfigForge auto-detects both formats:

```bash
devbench cf convert config.json config.yaml
```

This reads `config.json`, detects it as JSON, and writes `config.yaml` as clean YAML.

## Stdin Piping (Quick One-Offs)

For quick conversions without creating intermediate files, pipe data through stdin:

```bash
cat config.json | devbench cf --to yaml
```

Or with a heredoc for inline data:

```bash
devbench cf --to yaml <<< '{"name": "myapp", "version": "1.0.0"}'
```

## Handling Nested JSON Objects

JSON with deep nesting converts to cleanly indented YAML:

```bash
# Input: config.json with nested "database" object
devbench cf --to yaml < config.json
# Output:
# database:
#   host: localhost
#   port: 5432
#   credentials:
#     user: admin
#     password_encrypted: true
```

YAML's indentation-based structure is more readable for humans than deeply nested JSON braces. ConfigForge preserves the nesting exactly, producing valid, clean YAML.

## JSON Arrays to YAML Lists

JSON arrays become YAML list syntax naturally:

```bash
# Input: {"servers": ["web01", "web02", "db01"]}
devbench cf --to yaml < servers.json
# Output:
# servers:
#   - web01
#   - web02
#   - db01
```

## Batch Converting Many JSON Files to YAML

Need to convert an entire directory? ConfigForge's batch mode handles glob patterns and recursive directory traversal:

```bash
devbench cf convert "configs/*.json" --to yaml --batch
```

This converts every JSON file in `configs/` to `.yaml` in a single command, with per-file progress reporting.

## Frequently Asked Questions

**Does it preserve comments?** JSON has no comment syntax, so there are no comments to preserve from the input. But when converting *from* comment-supporting formats (YAML, TOML, INI, .env) to JSON, ConfigForge preserves those comments in the intermediate representation.

**Can I convert JSON to TOML?** Yes! ConfigForge handles JSON ↔ TOML bidirectionally. This is useful when migrating Node.js configs (JSON) to Rust Cargo.toml-style configs.

**Is it free?** Yes — `pip install devbench-cf` installs the full CLI for free. No trial, no premium tier, no feature gates.

**Is it really 100% offline?** Yes. ConfigForge makes zero external API calls. Your data never leaves your machine — not even for telemetry.
