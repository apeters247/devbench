## CRITICAL FINDINGS

### 1. Hardcoded Absolute Paths — scripts/run_ai.py
**Lines:** 25, 32, 65, 97, 119
- `/tmp/devbench_locks` — world-writable /tmp, symlink attack surface
- `/var/www/devbench` — hardcoded project root, not portable
- `/home/andrew/.nvm/versions/node/v22.22.0/bin/gemini` — user-home path
- `/var/www/devbench/scripts/ask_gemini.py` — hardcoded script path
- `/var/www/herbalist/.env` — hardcoded external project path

**Risk:** Breaks on any other machine; /tmp location exposes TOCTOU race.
**Fix:** Use `Path(__file__).resolve().parent` for relative paths; env vars for machine-specific config.

---

### 2. Credentials Read from Arbitrary .env File — scripts/run_ai.py
**Lines:** 119, 121-127
Reads `GOOGLE_API_KEY` from `/var/www/herbalist/.env` via string splitting. No masking, no secure deletion after use.

**Risk:** Key leaks in logs, error messages, debugging output.
**Fix:** Use `os.environ.get()` only; never read credentials from arbitrary files.

---

### 3. Subprocess with Unvalidated User Input — scripts/run_ai.py
**Lines:** 34-46, 71-77, 101-106
`prompt_text` passed directly as `-p prompt_text` argument to claude/gemini CLIs without sanitization.

**Risk:** Crafted prompt could exploit downstream CLI; process args visible in `ps aux`/`/proc/[pid]/cmdline`.
**Fix:** Validate/sanitize prompt_text; pass via stdin pipe rather than CLI arg.

---

### 4. Path Traversal TOCTOU — web/serve.py
**Lines:** 421-456
`_handle_demo()` checks symlinks after path resolution, but there is a race window between symlink check and file read (TOCTOU). Hard links bypass symlink check entirely.

**Risk:** Attacker-controlled symlink swapped to hard link between check and read could escape sandbox.
**Fix:** Use `os.path.realpath()` result and re-verify containment immediately before `open()`.

---

### 5. Uncontrolled File Write in Batch Operations — core/configforge.py
**Lines:** 2990-2991, 3060-3061
```python
out_path = Path(output_dir) / f"{in_path.stem}.{to_fmt}"
```
No validation that `output_dir` is within an allowed location; `mkdir(parents=True)` can create directories anywhere the process has permissions.

**Risk:** Writes files outside intended scope if output_dir is adversarial.
**Fix:** Reject paths with `..`; verify resolved path stays within a whitelist before writing.

---

### 6. Unsafe Unbounded Glob — core/configforge.py
**Lines:** 2973, 3045
```python
files = sorted(glob.glob(input_glob, recursive=recursive))
```
No limit on number of files matched; `recursive=True` allows `**` patterns spanning the entire filesystem.

**Risk:** DoS via memory exhaustion; potential traversal of sensitive directories.
**Fix:** Cap matched files at e.g. 10,000; warn on large matches; restrict glob base to explicit directory.

## HIGH FINDINGS

### 7. Race Condition in /tmp Lock Directory — scripts/run_ai.py
**Lines:** 25-26
`/tmp/devbench_locks` created without restrictive permissions (should be `0o700`). Attacker can pre-create it.

**Fix:** Use `tempfile.mkdtemp(mode=0o700)` or validate ownership before use.

---

### 8. Broad Exception Catches — web/api.py, web/serve.py
**api.py lines:** 260-261, 272-273 | **serve.py lines:** 474-476
`except Exception` catches `SystemExit`, `KeyboardInterrupt`, `MemoryError` — unrecoverable conditions that should propagate.

**Fix:** Catch specific exceptions; re-raise `BaseException` subclasses that aren't `Exception`.

---

### 9. Missing Input Size Pre-Validation — web/api.py
**Lines:** 206-211
`MAX_BODY_SIZE` check happens after reading headers. A `Content-Length: 0` + oversized body could slip through in edge cases.

**Fix:** Validate `Content-Length` header value before initiating body read.

---

### 10. Hardcoded Default Ports — web/serve.py:30, web/api.py:45, core/cli.py:320,324
Ports 8080 and 8081 hardcoded as defaults with no conflict detection.

**Fix:** Document defaults clearly; support `PORT=0` for auto-assignment in dev.
