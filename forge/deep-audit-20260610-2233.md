## MEDIUM

### M1 — Internal exception details leaked in 500 responses (`web/api.py:258,270`)

```python
self._send_json({"success": False, "error": "Internal error: %s" % e}, status=500)
```

Exception `str()` is sent verbatim to callers. On production deployments, this can expose file paths, internal module names, or stack hints. The 500 handler in `do_POST` and `do_GET` both have this pattern.

**Recommendation:** Replace with a generic message in production, or gate detail on a `--debug` flag.

## LOW

### L1 — User-controlled path reflected in 404 JSON response (`web/license_server.py:245`)

```python
self._respond(*_error(404, f"Unknown path: {path}"))
```

`path` comes from `urllib.parse.urlparse(self.path).path` and is echoed into the JSON error. Not XSS-exploitable (JSON API, not HTML), but unnecessarily reflects user input. Use a static message like `"Unknown endpoint"`.

### L2 — Port conflict error says `cf --serve` unconditionally (`core/cli.py:1273`)

The `_check_port_available()` function accepts a `command_hint` parameter and uses it correctly in the error message. However, the hint defaults to `"cf --serve"` — callers for `--api` and `--license` override it correctly. This is working as intended; no action needed.
