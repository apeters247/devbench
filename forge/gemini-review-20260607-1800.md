## 5. False Positives / Overstated Concerns

- **Deep CSV heuristics**: The `detect_format` CSV handler (L810-949) is verbose, but every heuristic is there for a good reason (CSV is inherently ambiguous). Calling it fragile is correct; calling it fixable with `python-magic` is a weak suggestion since mime type (`text/csv`) won't distinguish CSV-with-header from CSV-without.
- **Duplicate `batch_convert` functions**: Looking more carefully, the first (L1639) is a separate internal batch; the second (L1775) is the CLI-facing variant. They have different signatures. The naming is confusing but not strictly a bug.
- **CLI `--from` keyword conflict**: `from` is indeed a Python keyword, but `argparse` handles this via `dest="from_fmt"` correctly. This is cosmetic at best.
