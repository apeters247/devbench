## 3. Phase 2 — Rotating Spot Check

### `core/configforge.py` (lines 1-1000)

**Bugs Found by Gemini:**

1. **Line 20: Incorrect `HAS_XML` initialization** — initialized `True` but treated as optional import. If import fails, `HAS_XML` remains `True` until the `except` block at line 53-56. Should be `False` by default.

2. **Line 67: `_count_delims_outside_quotes` — escaped quotes not handled** — `\\"` or `\\'` incorrectly toggles `in_quotes` state, leading to miscount of delimiters inside quoted strings.

3. **Lines 82-84: `_yaml_find_comment` — escaped quotes within double-quoted strings** — `\\"` inside YAML double-quoted strings flips `in_double` flag incorrectly, causing `#` characters inside strings to be misidentified as comment starters.

4. **Line 124: `_build_key_paths` — quoted keys containing colons** — Splits on first colon without considering quotes. A key like `"my:key": value` gets truncated to `"my"` instead of preserving the full quoted key.

5. **Lines 169-171: Fragile list-item comment anchoring** — Uses only stripped scalar value; non-unique list item values cause ambiguous comment reinsertion.

6. **Lines 207, 223: Unused `inserted` variable and premature `inserted_set` initialization** — dead code.

7. **Lines 219-224: `path_to_lines` not shifted after standalone comment insertion** — After inserting standalone comments, line indices in `path_to_lines` are stale, causing `path_keyed` comment insertion to use wrong line numbers.

8. **Line 234: Misplaced import** — `from collections import OrderedDict` inside function body.

## 4. False Positives / Questionable Claims

- **`HAS_XML` initialization (Line 20)**: The init to `True` followed by a later `except` block is a pattern, but the gap between init and the `except` handler is real — if code between uses `HAS_XML` before the `except` fires, it would see `True` when XML is unavailable. However, in practice the `try/except` wraps the import immediately below, so this is a mild tech-debt concern, not a live bug.
- **`_count_delims_outside_quotes` escaped quotes (Line 67)**: Valid point for CSV-style delimiters. The actual usage context for this function in this codebase matters — if it's only used with YAML/TOML parsers that handle escaping at a higher level, this may not trigger in practice.
