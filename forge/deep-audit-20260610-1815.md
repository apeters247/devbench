## CRITICAL FINDINGS

### 1. YAML Import Error Handling in schema_infer (tools.py:1377-1379)
**File:** `core/tools.py` lines 1377-1379
- `schema_infer()` does a late `import yaml` when `output_format == "yaml"` with no error handling
- Everywhere else in the codebase (`configforge.py`) uses `HAS_YAML` guard to give a friendly error

- **Fix:** Add `HAS_YAML` guard in schema_infer matching the pattern used in configforge.py
