## 3. 🟡 High Issues (2)

### 🟡 HIGH-1: `_apply_select_filter` missing regex support — NEW
**File:** `core/cli.py:3367-3403`
**Status:** NEW this cycle — introduced when `_apply_select_filter` was extracted as a shared helper.
**Detail:** The main `_run_cf_select` (line 3234) correctly handles `/pat/` regex operators with full `re.compile()` logic (lines 3307-3328). But the extracted shared helper `_apply_select_filter` (called by `_run_cf_each` at line 3446 and `_run_cf_join` at line 3888) only handles `eq`, `neq`, `contains`, `not_contains` — no regex.

When `--each` or `--join` is combined with a regex select:
```
devbench cf pods.yaml --select status=/Running/ --each name
```
The `=` operator is detected, `raw_val = "/Running/"`, `_coerce_set_value("/Running/")` returns the string `"/Running/"`, and the filter does an exact equality check instead of a regex match. No error is raised — wrong results are silently returned (typically empty or spurious matches).

**Impact:** Users who combine `--select FIELD=/pat/` with `--each` or `--join` get silent wrong output. This directly breaks the advertised composition documented in the `--select` help string ("Combine with --get to navigate to the list first").
**Fix:** Add regex detection to `_apply_select_filter` — copy the 4-line `/pat/` detection block from `_run_cf_select`:
```python
import re as _re
regex_pat = None
if op in ("eq", "neq") and raw_val.startswith("/") and raw_val.endswith("/") and len(raw_val) >= 2:
    try:
        regex_pat = _re.compile(raw_val[1:-1], _re.IGNORECASE)
        op = "regex_match" if op == "eq" else "regex_not_match"
    except _re.error:
        return data  # graceful: invalid regex → no filter
coerced_val = _cf._coerce_set_value(raw_val) if regex_pat is None else None
```

**File:** `tests/test_license.py`
**Status:** Persistent from first audit (5 cycles without fix).
**Detail:** `_start_server` opens a socket on port 0, reads the bound port, closes the socket, then starts the server. A concurrent process can steal the port between close and start.

### 🟢 MEDIUM-4: `configforge.main()` now >10 features behind `cli.py` — WORSENING
**File:** `core/configforge.py` (bottom ~200 lines: `main()` function)
**Status:** Was 5 features behind at last audit (MEDIUM-4), now divergence is larger.
**Detail:** Since last audit, `cli.py` has gained: `--quiet`, `--join`, bracket-notation path access, `--each`+`--join` composition, `--list-merge merge`, `--wrap-in`, `--schema-gen`, `--replace-value`, `--sort-by`, `--unique`, `--unique-by`, `--sort-desc`, `--compact`, `--template`, `--select` (regex), `--default`, `--sort-keys-reverse`. **Fix:** Delete `configforge.main()` and remove the `configforge` console_scripts entry point from `setup.py`/`pyproject.toml`. All users should use `devbench cf`. The `configforge` CLI is a maintenance liability.
